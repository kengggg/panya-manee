# Full Benchmark Execution Checklist

**Scope:** Expanded NT P3 local benchmark on the Mac mini M4 16 GB baseline.
**Machine profile:** `macmini-m4-16gb-ollama`
**Eval split:** `text_only_core` only, 93 items total per run (Thai 60, Math 33)
**Compatibility gate:** `mcq_text_only_v1`, prompt `v1_answer_only`, scoring `v1`, think `off`

This runbook is **script-first**. The source of truth is repo workflows and scripts, not chat.

---

## 0. Hard Rules for Determinism

1. **Fixed dataset** only, `text_only_core`, no manual filtering or reordering.
2. **Fixed prompt path** only, whatever `config.py` and runtime scripts define. No ad-hoc prompt edits mid-batch.
3. **Fixed think mode** for V1, `off` only. No mixed think/non-think in the same benchmark line.
4. **Fixed batch IDs** before execution. Never rename or recycle a batch after outputs exist.
5. **Record model digests before verified publication.** If digests drift, the verified batch is invalid for publication.
6. **No manual cherry-picking.** Do not delete bad runs, exclude files by hand, or selectively publish winners outside the verification report.
7. **Main leaderboard only includes models that survive the verified post-hoc gate.**
8. **Run serially on the 16 GB Mac mini.** Never overlap benchmark batches.
9. **Use scripts/workflows for every round.** Human role = dispatch, verify, approve, and document decisions.
10. **Separate 3x screening is retired.** The verified batch itself now decides publication eligibility.

## 4. Phase 1, Preflight

Goal: verify that the batch can run deterministically before any official round starts.

### Checklist
- [ ] `uv sync --frozen` succeeds
- [ ] Ollama reachable at `http://localhost:11434`
- [ ] all candidate models are pre-pulled
- [ ] model digest record saved as machine-readable JSON artifact
- [ ] dataset integrity verified, Thai 60 + Math 33 for `text_only_core`
- [ ] working tree is clean on `main`
- [ ] any obviously non-fitting model is removed **before** smoke or verified publication

### Deterministic commands
```bash
uv sync --frozen

uv run python scripts/preflight.py \
  --models "$MODELS" \
  --pull
```

Output: `preflight_records/preflight_YYYYMMDD.json`

**Stop here if preflight exits non-zero.** Remove failing models from the active candidate list and document why.

## 5. Phase 2, Smoke Pass (1x, optional but recommended)

Goal: prove the expanded pipeline works end to end, not to make publication decisions.

### Batch ID
- `ntp3-smoke-r1-YYYYMMDD`

### Checklist
- [ ] dispatch one real pass across all practical candidates
- [ ] verify raw response files exist for each attempted model
- [ ] verify repeat summary exists
- [ ] dry-run snapshot validation passes
- [ ] note any operational failures

### Preferred execution path
```bash
gh workflow run benchmark-run.yml \
  -f batch_id="ntp3-smoke-r1-$(date +%Y%m%d)" \
  -f models="$MODELS" \
  -f runs_per_model="1" \
  -f subjects="thai,math" \
  -f think_budget="" \
  -f dry_run=false
```

Then validate the batch deterministically:

```bash
uv run python scripts/publish_snapshot.py --batch-id "ntp3-smoke-r1-$(date +%Y%m%d)" --dry-run --allow-unverified-publication
```

Smoke output is operational evidence only. It must not be treated as a ranking or publication basis.

## 6. Phase 3, Verified Publication Batch (2x on all candidates)

Goal: run the real benchmark twice for every surviving model, verify canonical vs shadow determinism, then publish only survivors from the verified batch.

### Batch ID
- `ntp3-vr1-YYYYMMDD`

### Checklist
- [ ] dispatch verified batch for all surviving candidates
- [ ] run exactly 2 repeats per model
- [ ] verify completeness of JSONL outputs via `check_batch_completeness.py`
- [ ] generate `verification_report_{batch_id}.json`
- [ ] confirm publishable survivors exist before attempting snapshot publication

### Execution
Preferred: use the dedicated verified workflow.

```bash
gh workflow run benchmark-verified.yml \
  -f batch_id="ntp3-vr1-$(date +%Y%m%d)" \
  -f models="$MODELS" \
  -f preflight_date="$(date +%Y%m%d)" \
  -f dry_run=false
```

### Publication gates inside the verified batch
A model is publishable only if all are true:
- [ ] canonical and shadow match item-by-item
- [ ] canonical parse rate **>= 95%**
- [ ] canonical accuracy **> 25%**
- [ ] model digest still matches preflight record, when preflight verification is available

### Explicit non-gate
- [ ] latency is **not** a hard exclusion gate, but it must be published as metadata

---

## 7. Phase 4, Snapshot Publication

Goal: build the dashboard snapshot from the verified batch and include only publishable survivors.

### Checklist
- [ ] `verification_report_{batch_id}.json` has `status: pass`
- [ ] `publishable_count >= 1`
- [ ] dry-run snapshot validation passes
- [ ] open snapshot PR from `site/data/latest/` + `registry/snapshots.json`

### Deterministic command
```bash
gh workflow run snapshot-pr.yml -f batch_id="ntp3-vr1-$(date +%Y%m%d)"
```

`snapshot-pr.yml` will refuse unverified hobby/test batches unless `publish_snapshot.py` is explicitly run with `--allow-unverified-publication` for local smoke use.

## 8. Manual Fallback Commands

If workflow dispatch is unavailable, use these exact script calls.

### Verify batch completeness
```bash
uv run python scripts/check_batch_completeness.py \
  --batch-id "ntp3-vr1-$(date +%Y%m%d)" \
  --models "$MODELS" \
  --runs-per-model 2
```

### Apply verification gate
```bash
uv run python scripts/verification_gate.py \
  --batch-id "ntp3-vr1-$(date +%Y%m%d)" \
  --expected-models "$MODELS" \
  --expected-runs 2 \
  --canonical-run-index 1 \
  --shadow-run-index 2 \
  --preflight "preflight_records/preflight_$(date +%Y%m%d).json" \
  --json-out "benchmark_responses/verification_report_ntp3-vr1-$(date +%Y%m%d).json"
```

### Dry-run snapshot build
```bash
uv run python scripts/publish_snapshot.py --batch-id "ntp3-vr1-$(date +%Y%m%d)" --dry-run
```

## 9. Migration Note, screen -> verified-only

Old shape:
- preflight
- screening 3x on all candidates
- promotion decision
- verified publication 2x on promoted models
- snapshot PR

New shape:
- preflight
- optional smoke
- verified publication 2x on all candidates
- post-hoc filtering from the verified report
- snapshot PR

Why the change:
- screening used the **same benchmark** as publication
- screening saved very few publication runs while costing many extra runs
- screening scores were mostly discarded instead of reused directly
- deterministic verification is the real publication safeguard, not a lenient 3-run gate

## 10. Source of Truth Files

3. `.github/workflows/benchmark-verified.yml` — verified-only publication workflow, 2x on all candidates
4. `scripts/check_batch_completeness.py` — deterministic file and row-count completeness check

6. `scripts/publish_snapshot.py` — publication gate plus snapshot build/validate/sync
7. `.github/workflows/snapshot-pr.yml` — PR opener for validated snapshot output
8. `registry/compatibility.json` — V1 contract values

`benchmark-screen.yml` is retained only as a deprecated stop sign and must not be used.
